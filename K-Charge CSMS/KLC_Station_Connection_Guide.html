2. Bước 1: Đăng Ký Trạm Trong CSMS

1 Tạo Charging Station trên Admin Portal

Trước khi charger kết nối, phải register trong hệ thống KLC CSMS. Nếu charger kết nối mà chưa đăng ký, CSMS sẽ trả về BootNotification.status = "Pending" và charger không thể hoạt động.

2.1. Qua Admin Portal (UI)

  1. Đăng nhập Admin Portal: https://admin.odcall.com
  2. Menu: Quản Lý Trạm Sạc → Thêm Trạm Mới
  3. Điền thông tin:
    • ChargePointId: KLC-HCM-001 (phải khớp chính xác với cấu hình trên charger)
    • Tên: Trạm sạc Quận 1
    • Vendor: Chargecore
    • Model: AC003+
    • Địa chỉ + GPS: ...
  4. Thêm Connectors:
    • Connector 1: Type2, Max 7kW
    • Connector 2: Type2, Max 7kW
  5. Lưu → Trạm ở trạng thái "Registered - Awaiting Connection"

5. Bước 3: Kiểm Tra Kết Nối

5.1. Verification Checklist

Sau khi cấu hình charger và đã register trong CSMS, thực hiện kiểm tra từng bước:

1 Kiểm Tra Network Connectivity

  • Charger có IP address (check trên charger screen hoặc web interface)
  • Từ charger network, ping được csms.odcall.com
  • Firewall cho phép outbound port 443 (WSS) hoặc 8081 (WS dev)
  • DNS resolves csms.odcall.com đúng IP

2 Kiểm Tra WebSocket Connection

  • Restart charger (power cycle hoặc soft reset)
  • Quan sát CSMS logs: phải thấy WebSocket connection từ charger
  • Quan sát CSMS logs: phải thấy BootNotification message
  • BootNotification response phải là "status": "Accepted"
  • Charger hiển thị "Connected" hoặc "Online" trên screen

CSMS Log mẫu (expected):

[2026-03-23 10:00:01 INF] WebSocket connection accepted: KLC-HCM-001
[2026-03-23 10:00:01 INF] BootNotification from KLC-HCM-001:
    Vendor=Chargecore, Model=AC003+, FW=1.2.0
[2026-03-23 10:00:01 INF] BootNotification response: Accepted, interval=300
[2026-03-23 10:00:01 INF] StatusNotification from KLC-HCM-001:
    Connector=0, Status=Available, Error=NoError
[2026-03-23 10:00:01 INF] StatusNotification from KLC-HCM-001:
    Connector=1, Status=Available, Error=NoError

3 Kiểm Tra Heartbeat

  • Đợi 5 phút (HeartbeatInterval = 300s)
  • CSMS logs phải thấy Heartbeat message
  • Admin Portal: trạm hiển thị status "Online"
  • Admin Portal: Last Heartbeat timestamp cập nhật đúng

4 Kiểm Tra Status Notification

  • Admin Portal: tất cả connectors hiển thị "Available" (xanh)
  • Cắm cáp sạc vào connector 1 (không cần xe) → Status chuyển "Preparing"
  • Rút cáp → Status chuyển lại "Available"
  • Nếu charger có error simulation → test Faulted status

5 Kiểm Tra Charging Session (End-to-End)

  • Chuẩn bị: 1 xe điện hoặc EV simulator + RFID card đã register
  • Test RFID: Quẹt RFID → CSMS nhận Authorize → Accepted → StartTransaction
  • Test Remote Start: Từ Admin Portal, gửi RemoteStartTransaction → Charger bắt đầu sạc
  • Quan sát MeterValues: CSMS nhận energy readings mỗi 60 giây
  • Dừng sạc (rút cáp hoặc RemoteStop) → StopTransaction → Session completed
  • Kiểm tra session trong Admin Portal: thời gian, energy (kWh), cost

5.2. CSMS Configuration Sync (Auto-Configure After Boot)

Sau khi BootNotification được Accepted, CSMS tự động gửi chuỗi lệnh cấu hình:

// Auto-configuration sequence triggered by BootNotification Accepted
async Task AutoConfigureChargerAsync(string chargePointId)
{
    var configs = new Dictionary<string, string>
    {
        ["HeartbeatInterval"] = "300",
        ["MeterValueSampleInterval"] = "60",
        ["ClockAlignedDataInterval"] = "900",
        ["MeterValuesSampledData"] =
            "Energy.Active.Import.Register,Power.Active.Import,Current.Import,Voltage,SoC",
        ["StopTxnSampledData"] = "Energy.Active.Import.Register",
        ["ConnectionTimeOut"] = "60",
        ["StopTransactionOnEVSideDisconnect"] = "true",
        ["StopTransactionOnInvalidId"] = "true",
        ["AuthorizeRemoteTxRequests"] = "true",
        ["LocalPreAuthorize"] = "true",
        ["LocalAuthorizeOffline"] = "true",
    };

    foreach (var (key, value) in configs)
    {
        try
        {
            var result = await _dispatcher.SendCommandAsync(
                chargePointId, "ChangeConfiguration",
                new JObject { ["key"] = key, ["value"] = value });

            var status = result.Value<string>("status");
            _logger.LogInformation(
                "Config {Key}={Value} → {Status} for {CP}",
                key, value, status, chargePointId);

            if (status == "RebootRequired")
                _logger.LogWarning("Config {Key} requires reboot for {CP}",
                    key, chargePointId);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Failed to set config {Key} for {CP}",
                key, chargePointId);
        }

        await Task.Delay(200); // Small delay between commands
    }
}

- DNS không resolve được

- ChargePointId không khớp 		- Kiểm tra ChargePointId trên charger vs CSMS (case-sensitive!)
- Register trạm trong Admin Portal trước
- Check CSMS logs cho warning message
Kết nối bị ngắt liên tục (reconnect loop) - Network không ổn định
- WebSocket ping/pong timeout
- CSMS reject connection 		- Kiểm tra bandwidth/latency
- Tăng WebSocket KeepAliveInterval
- Check CSMS logs cho disconnect reason
- JUHANG 4G Check signal strength
MeterValues không gửi về - MeterValueSampleInterval = 0 (disabled)
- MeterValuesSampledData trống
- Charger không hỗ trợ measurand 		- Dùng GetConfiguration verify giá trị hiện tại
- Set lại qua ChangeConfiguration
- Thử từng measurand riêng để xác định unsupported
RemoteStartTransaction bị Rejected - Connector không Available
- AuthorizeRemoteTxRequests = true nhưng idTag invalid
- ConnectorId sai 		- Verify connector status = Available
- Verify idTag tồn tại và valid trong CSMS
- Verify connectorId (1-based, không phải 0)
JUHANG 4G không kết nối - APN sai
- SIM chưa activate
- Signal yếu 		- Verify APN theo nhà mạng (xem bảng 4.3)
- Gọi nhà mạng activate data service
- Di chuyển antenna hoặc thử vị trí khác
- Check SIM balance (data quota)
Connector stuck ở "Preparing" - ConnectionTimeOut quá lớn
- EV không respond to pilot signal 		- Set ConnectionTimeOut = 60
- Check cáp sạc / connector vật lý
- Soft reset charger
Charger hiển thị Faulted - Hardware error (ground fault, overcurrent, etc.)
- Communication error 		- Check StatusNotification errorCode trong CSMS logs
- Kiểm tra hệ thống điện (earth, phase)
- Hard reset charger
- Liên hệ vendor support nếu lỗi hardware

7.3. ALB Configuration for WebSocket

Quan trọng: AWS ALB hỗ trợ WebSocket natively nhưng cần cấu hình:
  • Idle Timeout: Set ≥ 600 seconds (10 minutes) - vì OCPP connection là persistent, HeartbeatInterval = 300s
  • Stickiness: Enable session stickiness (duration-based, 1 day) - đảm bảo charger luôn route đến cùng instance
  • Health Check: GET /health trên port 5000, interval 30s
  • Target Group: Protocol = HTTP (ALB terminates TLS), Port = 5000

8. Checklist Commissioning Trạm Sạc Mới

Sử dụng checklist này cho mỗi trạm sạc mới được lắp đặt. In ra và đánh dấu tại hiện trường.

Giai đoạn 1: Chuẩn bị (Trước khi ra hiện trường)

  • ChargePointId đã được assign: KLC-____-____
  • Trạm đã register trong CSMS Admin Portal
  • Connectors đã thêm đúng số lượng và loại
  • Tariff plan đã gán cho trạm
  • RFID test card sẵn sàng (đã register trong hệ thống)
  • Thông tin WiFi/4G SIM sẵn sàng
  • Laptop với trình duyệt web sẵn sàng

Giai đoạn 2: Lắp đặt vật lý

  • Charger lắp đặt đúng vị trí, cố định an toàn
  • Hệ thống điện kết nối đúng (kiểm tra phase, earth)
  • Cáp Ethernet kết nối (nếu dùng Ethernet)
  • SIM card lắp đúng (nếu dùng 4G)
  • Charger bật nguồn thành công, màn hình hiển thị

Giai đoạn 3: Cấu hình phần mềm

  • Truy cập web interface charger thành công
  • OCPP URL đã cấu hình đúng: wss://csms.odcall.com/ocpp/KLC-____-____
  • OCPP version = 1.6J (JSON)
  • ChargePointId khớp giữa charger config, URL, và CSMS registration
  • Network (Ethernet DHCP / 4G APN) cấu hình đúng
  • Authentication credentials (nếu có) cấu hình đúng
  • Đổi mật khẩu admin mặc định của charger

Giai đoạn 4: Kiểm tra kết nối

  • Charger kết nối CSMS thành công (BootNotification → Accepted)
  • Trạm hiển thị "Online" trên Admin Portal
  • Tất cả connectors hiển thị "Available"
  • Heartbeat hoạt động (đợi 5 phút, verify trên CSMS)
  • CSMS auto-configuration hoàn tất (check logs)

Giai đoạn 5: Test charging session

  • Test RFID authorize → Accepted
  • Cắm xe / EV simulator → StartTransaction
  • MeterValues đang gửi về CSMS (check mỗi 60 giây)
  • Admin Portal hiển thị real-time charging data
  • Dừng sạc → StopTransaction → Session completed
  • Session history hiển thị đúng: thời gian, kWh, cost
  • Test Remote Start từ Admin Portal → charger bắt đầu sạc
  • Test Remote Stop từ Admin Portal → charger dừng sạc

Giai đoạn 6: Hoàn tất

  • Chụp ảnh charger đã lắp đặt
  • Ghi lại Serial Number, Firmware Version từ BootNotification
  • Cập nhật vị trí GPS chính xác trên Admin Portal
  • Test ngắt điện → bật lại → charger tự reconnect
  • Ghi nhận kết quả commissioning vào hệ thống
  • Trạm chuyển trạng thái "Operational" trên Admin Portal
Trạm sạc đã sẵn sàng phục vụ! Từ lúc này, trạm sẽ tự động gửi heartbeat, status updates, và meter data về CSMS. Admin có thể monitor real-time qua Portal.
